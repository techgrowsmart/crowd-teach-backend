3. COMPONENT INTERACTIONS

┌─────────────────────────────────────────────────────────────────────────────────────────┐ │ USER REGISTRATION FLOW │ └─────────────────────────────────────────────────────────────────────────────────────────┘ 1. USER → APP: Enter email & details 2. APP → API: POST /api/signup 3. API → RATE LIMIT: Check email/IP limits 4. API → DATABASE: Check existing user 5. API → EMAIL SERVICE: Generate & send OTP 6. API → DATABASE: Store OTP with expiration 7. API → APP: Return OTP ID 8. USER → APP: Enter OTP 9. APP → API: POST /api/signup/verify-otp 10. API → DATABASE: Verify OTP 11. API → DATABASE: Create user record 12. API → DATABASE: Create student/teacher record 13. API → REDIS: Cache user data 14. API → APP: Return JWT token 15. APP → STORAGE: Store token securely ┌─────────────────────────────────────────────────────────────────────────────────────────┐ │ CLASS BOOKING FLOW │ └─────────────────────────────────────────────────────────────────────────────────────────┘ 1. USER → APP: Browse classes 2. APP → API: GET /api/classes (with auth) 3. API → CACHE: Check cached classes 4. API → DATABASE: Fetch classes if cache miss 5. API → CACHE: Store classes data 6. API → APP: Return classes list 7. USER → APP: Select class & book 8. APP → API: POST /api/book-class (with auth) 9. API → AUTH: Verify JWT token 10. API → DATABASE: Check availability 11. API → DATABASE: Create booking record 12. API → PAYMENT: Initiate Razorpay payment 13. PAYMENT → API: Payment confirmation 14. API → DATABASE: Update booking status 15. API → NOTIFICATION: Send confirmation 16. API → APP: Return booking confirmation
