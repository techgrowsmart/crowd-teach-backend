WEBSOCKET CONFIGURATION
================================================================================

The WebSocket runs on the SAME PORT as HTTP API (3000):

Frontend Connection:
  Development: ws://localhost:3000
  Production:  wss://growsmartserver.gogrowsmart.com

Key Points:
  ✅ No separate port needed
  ✅ SSL terminates at CloudFront/ALB
  ✅ Socket.IO handles upgrade automatically
  ✅ CORS configured for all gogrowsmart.com subdomains

================================================================================
DEPLOYMENT STEPS (EC2)
================================================================================

1. SSH into EC2:
   ssh -i your-key.pem ec2-user@growsmartserver.gogrowsmart.com

2. Navigate to backend directory:
   cd crowd-teach-gogrowsmart-backend

3. Run deployment:
   ./deploy-ec2-docker.sh

4. Verify deployment:
   curl http://localhost:3000/api/ping

5. For SSL (Let's Encrypt):
   sudo certbot certonly --standalone -d growsmartserver.gogrowsmart.com
   sudo cp /etc/letsencrypt/live/growsmartserver.gogrowsmart.com/*.pem ./certs/

================================================================================
DATABASES CONFIGURED
================================================================================

1. AstraDB (Cassandra) - PRIMARY
   - Cloud-hosted DataStax Astra
   - Keyspace: teachnteachprod
   - Used for: Core application data

2. MongoDB Atlas - POSTS/GROWTHOUGHTS
   - Cloud-hosted MongoDB
   - Database: GrowThoughts
   - Used for: Posts, thoughts, social features

3. Redis - LOCAL (Docker)
   - Runs in Docker container
   - Port: 6379 (internal)
   - Used for: Caching, sessions, real-time features

# Redis CLI:
docker-compose -f docker-compose.production.yml exec redis redis-cli

# Restart:
docker-compose -f docker-compose.production.yml restart

# Stop:
docker-compose -f docker-compose.production.yml down
